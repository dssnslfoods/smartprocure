## 1. ภาพรวมระบบ

**Smart Procurement** คือระบบจัดซื้อจัดจ้างแบบครบวงจรสำหรับ NSL Foods PLC ที่ออกแบบมาเพื่อบริหารจัดการกระบวนการจัดซื้อตั้งแต่ต้นจนจบ ได้แก่:

- **บริหาร Supplier** — ลงทะเบียน ตรวจสอบ และจัดการข้อมูล Supplier
- **ออก RFQ** — จัดทำใบขอราคาและเชิญ Supplier เสนอราคา
- **ประมูลออนไลน์** — จัดการประมูลย้อนกลับ (Reverse Auction) แบบ Real-time
- **เปรียบเทียบใบเสนอราคา** — เปรียบเทียบและคัดเลือก Supplier ที่ดีที่สุด
- **ตัดสินใจสั่งซื้อ** — บันทึกและอนุมัติการตัดสินใจจัดซื้อ
- **ประเมิน Supplier** — ให้คะแนนและเปรียบเทียบประสิทธิภาพ Supplier
- **รายงาน** — วิเคราะห์ค่าใช้จ่ายและผลการจัดซื้อ

## 3. การเข้าสู่ระบบ

### 3.1 เข้าสู่ระบบสำหรับผู้ใช้ทั่วไป

1. เปิดเบราว์เซอร์ไปที่ URL ของระบบ
2. กรอก **Email** และ **รหัสผ่าน**
3. กดปุ่ม **Sign in**
4. ระบบจะพาไปที่หน้า **Dashboard** โดยอัตโนมัติ

> **หมายเหตุ:** หากยังไม่มีบัญชี ให้ติดต่อผู้ดูแลระบบ หรือหากเป็น Supplier ใหม่ ให้กดลิงก์ **"ลงทะเบียน Supplier"** ที่หน้าเข้าสู่ระบบ

### 3.2 สำหรับ Supplier ที่เพิ่งลงทะเบียน

- หลังส่งข้อมูลลงทะเบียน ระบบจะ **ออกจากระบบให้อัตโนมัติ**
- ต้องรอผู้ดูแลระบบ **อนุมัติ** ก่อนจึงจะเข้าสู่ระบบได้
- เมื่อได้รับการอนุมัติแล้ว จะได้รับการแจ้งเตือน (ตามการตั้งค่าอีเมล)

### 3.3 ออกจากระบบ

- คลิกที่ไอคอน **ออกจากระบบ (LogOut)** ที่ด้านล่างซ้ายของแถบเมนู

## 4. หน้าแดชบอร์ด

หน้าแดชบอร์ดแสดงภาพรวมสำคัญของระบบ เข้าถึงได้จากเมนู **Dashboard**

### 4.1 การ์ดสรุปสถิติ (KPI Cards)

| การ์ด | ความหมาย |
|---|---|
| **Total Suppliers** | จำนวน Supplier ทั้งหมด (อนุมัติแล้ว + รอดำเนินการ) |
| **Open RFQs** | จำนวน RFQ ที่อยู่ในสถานะ Draft |
| **Active Auctions** | จำนวนการประมูลที่กำลังดำเนินอยู่ |
| **Pending Awards** | จำนวนการตัดสินใจสั่งซื้อที่รอการอนุมัติ |

> คลิกที่การ์ดเพื่อไปยังหน้าที่เกี่ยวข้องได้โดยตรง

### 4.2 ส่วนแสดงผลเพิ่มเติม

- **Suppliers by Status** — แสดงสัดส่วน Supplier แต่ละสถานะในรูปแบบ Progress Bar
- **Recent Activity** — แสดงกิจกรรมล่าสุดของระบบ (การลงทะเบียน Supplier, สร้าง RFQ, เปิดประมูล)
- **Top Rated Suppliers** — อันดับ Supplier ที่มีคะแนนสูงสุด พร้อมเหรียญ 🥇🥈🥉

### 4.3 สีแสดงสถานะทั่วไป

| สถานะ | สีที่แสดง | ความหมาย |
|---|---|---|
| Draft / แบบร่าง | เทา | ยังไม่ส่ง |
| Submitted / รอตรวจสอบ | น้ำเงิน | ส่งแล้ว รอดำเนินการ |
| Review / กำลังตรวจสอบ | เหลือง | อยู่ระหว่างตรวจสอบ |
| Approved / อนุมัติแล้ว | เขียว | ผ่านการอนุมัติ |
| Rejected / ถูกปฏิเสธ | แดง | ไม่ผ่านการอนุมัติ |
| Suspended / ถูกระงับ | ส้ม | ถูกระงับการใช้งาน |

## 5. การจัดการ Supplier

เมนู **Suppliers** ใช้สำหรับดูและบริหารข้อมูล Supplier ทั้งหมดในระบบ (สำหรับ Admin, เจ้าหน้าที่จัดซื้อ, ผู้อนุมัติ, ผู้บริหาร)

### 5.1 รายชื่อ Supplier

**วิธีค้นหา Supplier:**
1. พิมพ์ชื่อบริษัทหรือเลขภาษีในช่อง **Search**
2. กรองตาม **สถานะ** (Status) จาก Dropdown
3. กรองตาม **Tier** จาก Dropdown

**คอลัมน์ในตาราง:**
- **Company** — ชื่อบริษัท (คลิกเพื่อดูรายละเอียด)
- **Tax ID** — เลขประจำตัวผู้เสียภาษี
- **Status** — สถานะปัจจุบัน
- **Tier** — ระดับ Supplier (Critical Tier 1 / Non-Critical Tier 1)
- **Created** — วันที่เพิ่มในระบบ

**เพิ่ม Supplier ใหม่** (Admin / เจ้าหน้าที่จัดซื้อ):
1. กดปุ่ม **"+"** มุมบนขวา
2. กรอกข้อมูลในฟอร์ม
3. กดปุ่ม **"Create Supplier"**

### 5.2 รายละเอียด Supplier

คลิกชื่อบริษัทจากรายชื่อเพื่อดูรายละเอียด

**แท็บต่างๆ:**

#### แท็บ Information (ข้อมูลทั่วไป)
- ข้อมูลบริษัท: ชื่อ, Tax ID, Email, โทรศัพท์, เว็บไซต์, Tier
- ที่อยู่: ที่อยู่, จังหวัด, ประเทศ
- สถานะพิเศษ: Preferred Supplier, Blacklisted
- หมายเหตุ

#### แท็บ Contacts (ผู้ติดต่อ)
- รายชื่อผู้ติดต่อของบริษัทนั้น
- ตำแหน่ง, อีเมล, โทรศัพท์
- ป้าย "Primary" แสดงผู้ติดต่อหลัก

#### แท็บ Documents (เอกสาร)
- รายการเอกสารที่อัปโหลด
- ชื่อไฟล์, ประเภท, ขนาดไฟล์
- ดาวน์โหลดเอกสารได้โดยตรง

#### แท็บ ESG Profile (โปรไฟล์ ESG)
- คะแนน Environmental, Social, Governance (แต่ละด้าน 0–100)
- ระดับความเสี่ยง: Low / Medium / High
- สถานะ Compliance: Pending / Compliant / Non-Compliant

### 5.3 การเปลี่ยนสถานะ Supplier

| สถานะปัจจุบัน | ปุ่มที่ใช้ | สถานะถัดไป |
|---|---|---|
| Draft | Send to Review | Submitted |
| Submitted | Start Review | Review |
| Review | Approve | Approved |
| Review | Reject | Rejected |
| Rejected | Return to Draft | Draft |

> **สิทธิ์:** Admin, เจ้าหน้าที่จัดซื้อ, และผู้อนุมัติ สามารถเปลี่ยนสถานะได้

### 5.4 แก้ไขข้อมูล Supplier

1. เข้าหน้ารายละเอียด Supplier
2. กดปุ่ม **Edit** มุมบนขวา
3. แก้ไขข้อมูลที่ต้องการ
4. กด **"Update Supplier"**

**ข้อมูลที่แก้ไขได้:**
- ชื่อบริษัท, Tax ID, Email, โทรศัพท์, เว็บไซต์
- ที่อยู่, จังหวัด, ประเทศ
- Tier (Critical Tier 1 / Non-Critical Tier 1)
- หมายเหตุ
- Preferred Supplier (สลับเปิด/ปิด)
- Blacklisted (สลับเปิด/ปิด)

## 7. Supplier Portal

สำหรับ **Supplier** ที่ได้รับการอนุมัติแล้ว เข้าใช้งานผ่านเมนู **"Supplier Portal"**

### 7.1 หน้าหลัก Supplier Portal

แสดงการ์ดสรุป 4 ใบ:
- **บริษัท** — ชื่อบริษัท
- **ผู้ติดต่อ** — จำนวนผู้ติดต่อ
- **เอกสาร** — จำนวนเอกสาร
- **Tax ID** — เลขภาษี

พร้อมป้ายแสดงสถานะการอนุมัติ

---

### 7.2 แท็บ: ข้อมูลบริษัท

แก้ไขข้อมูลบริษัทได้โดยตรง:
1. แก้ไขข้อมูลที่ต้องการ
2. กด **"บันทึกข้อมูล"**

---

### 7.3 แท็บ: ผู้ติดต่อ

**เพิ่มผู้ติดต่อใหม่:**
1. กด **"เพิ่มผู้ติดต่อ"**
2. กรอก: ชื่อ-นามสกุล, ตำแหน่ง, อีเมล, เบอร์โทร
3. กด **"บันทึก"**

**แก้ไข/ลบผู้ติดต่อ:** กดไอคอนดินสอหรือถังขยะในแต่ละรายการ

## 8. การออก RFQ

**RFQ (Request for Quotation)** คือใบขอราคาที่ส่งให้ Supplier เสนอราคาสินค้าหรือบริการ

เมนู **RFQ** — สำหรับ Admin และเจ้าหน้าที่จัดซื้อ

### 8.1 ขั้นตอนการทำ RFQ

```
Draft → Published → Closed → Evaluation → Awarded
```

| ขั้นตอน | ความหมาย | ปุ่มที่ใช้ |
|---|---|---|
| Draft | กำลังร่าง ยังไม่เปิดรับราคา | Publish RFQ |
| Published | เปิดรับราคาจาก Supplier | Close Submissions |
| Closed | ปิดรับราคาแล้ว กำลังประเมิน | Start Evaluation |
| Evaluation | กำลังประเมินใบเสนอราคา | Mark Awarded |
| Awarded | ตัดสินใจแล้ว | — |

---

### 8.2 สร้าง RFQ ใหม่

1. ไปที่เมนู **RFQ**
2. กดปุ่ม **"Create RFQ"**
3. กรอกข้อมูล:

**ข้อมูลทั่วไป:**
| ฟิลด์ | บังคับ | คำอธิบาย |
|---|:---:|---|
| Title | ✅ | ชื่อ RFQ เช่น "RFQ สินค้า A ไตรมาส 2/2569" |
| Description | | รายละเอียดความต้องการ |
| Deadline | | วันและเวลาปิดรับราคา |
| Notes | | หมายเหตุเพิ่มเติม |

**รายการสินค้า/บริการ (Line Items):**
กด **"+ Add Item"** เพื่อเพิ่มรายการ

| ฟิลด์ | บังคับ | คำอธิบาย |
|---|:---:|---|
| Item Name | ✅ | ชื่อสินค้า/บริการ |
| Quantity | | จำนวน |
| Unit | | หน่วย เช่น กก., ชิ้น, ลัง |
| Specifications | | ข้อกำหนดเพิ่มเติม |

4. กด **"Create RFQ"** — ระบบจะสร้างหมายเลข RFQ อัตโนมัติ (เช่น RFQ-1714023600000)

---

### 8.3 เชิญ Supplier

หลังสร้าง RFQ แล้ว ต้องเชิญ Supplier ก่อน Publish:

1. เข้าหน้ารายละเอียด RFQ
2. คลิกแท็บ **"Invited Suppliers"**
3. **คอลัมน์ขวา:** ค้นหา Supplier ที่อนุมัติแล้ว → ติ๊กเลือก → กด **"Invite (X)"**
4. Supplier ที่เชิญแล้วจะย้ายมาแสดงใน **คอลัมน์ซ้าย**
5. ลบ Supplier ที่เชิญแล้วได้ด้วยปุ่ม Remove (เฉพาะตอน Draft)

> **หมายเหตุ:** ต้องเชิญ Supplier อย่างน้อย 1 ราย จึงจะ Publish RFQ ได้

---

### 8.4 รับและดูใบเสนอราคา

1. เข้าหน้ารายละเอียด RFQ
2. คลิกแท็บ **"Quotations"**
3. กด **"Submit Quotation"** เพื่อป้อนราคาจาก Supplier

**ข้อมูลในการป้อนใบเสนอราคา:**
- เลือก Supplier
- ราคาต่อหน่วยแต่ละรายการ
- สกุลเงิน (USD / ETB / EUR)
- ระยะเวลาคุ้มครองราคา (วัน)
- เงื่อนไขชำระเงิน เช่น Net 30
- เงื่อนไขการจัดส่ง เช่น FOB, CIF

## 9. ระบบ e-Bidding

ระบบ **e-Bidding (Electronic Bidding)** หรือการประมูลย้อนกลับ (Reverse Auction) ช่วยให้ Supplier แข่งกันเสนอราคาต่ำสุดแบบ Real-time

เมนู **e-Bidding**

### 9.1 ขั้นตอนการประมูล

```
Scheduled (กำหนดการ) → Active (กำลังประมูล) → Closed (ปิดแล้ว)
```

---

### 9.2 สร้างงานประมูลใหม่

1. ไปที่เมนู **e-Bidding**
2. กด **"New Auction"**
3. กรอกข้อมูล:

| ฟิลด์ | บังคับ | คำอธิบาย |
|---|:---:|---|
| Title | ✅ | ชื่องานประมูล |
| Linked RFQ | | เชื่อมกับ RFQ ที่ Published (ถ้ามี) |
| Start Time | ✅ | วันเวลาเริ่มประมูล |
| End Time | ✅ | วันเวลาสิ้นสุด |
| Max Rounds | | จำนวนรอบสูงสุด |
| Description | | รายละเอียด |

4. กด **"Create Event"**

---

### 9.3 การจัดการงานประมูล (สำหรับ Admin / เจ้าหน้าที่จัดซื้อ)

**เปิดประมูล:**
- กดปุ่ม **▶ Start Auction** เพื่อเปลี่ยนสถานะเป็น Active

**ระหว่างประมูล:**
- กด **"Next Round"** เพื่อขึ้นรอบถัดไป
- กด **"Close Auction"** เพื่อปิดก่อนกำหนด (สีแดง)

---

### 9.4 หน้ารายละเอียดงานประมูล

**การ์ดสถิติ Real-time:**
| การ์ด | ความหมาย |
|---|---|
| Time Left | เวลาที่เหลือ (ชั่วโมง:นาที:วินาที) |
| Round | รอบปัจจุบัน / รอบสูงสุด |
| Lowest Bid | ราคาเสนอต่ำสุดในขณะนั้น |
| Total Bids | จำนวนการเสนอราคาทั้งหมด |

**แท็บ Live Ranking:**
- อันดับ Supplier จากราคาต่ำสุด
- อันดับ 1 แสดงพื้นหลังสีเขียว พร้อมไอคอน 🏆
- อัปเดตแบบ Real-time

**แท็บ Submit Bid (สำหรับ Supplier / Admin):**
1. เลือก Supplier
2. กรอกจำนวนเงิน
3. กด **"Submit Bid"**

**แท็บ All Bids History:**
- ประวัติการเสนอราคาทั้งหมด แยกตามรอบ
- แสดงรอบ (R1, R2, ...), ชื่อ Supplier, ราคา, เวลา

## 13. การประเมิน Supplier

ใช้ประเมินประสิทธิภาพการทำงานของ Supplier โดยใช้เกณฑ์ที่กำหนดไว้

เมนู **Evaluations** — สำหรับ Admin และเจ้าหน้าที่จัดซื้อ

### 13.1 แท็บ: Evaluations

แสดงรายการการประเมินทั้งหมด

**ปุ่มดำเนินการ:**
- **Compare** — เปรียบเทียบคะแนน Supplier
- **New Template** — สร้างเกณฑ์ประเมินใหม่
- **Score Supplier** — ประเมิน Supplier

**สีของคะแนน:**
| ช่วงคะแนน | สี | ความหมาย |
|---|---|---|
| 4.0 – 5.0 | เขียว | ดีเยี่ยม |
| 3.0 – 3.9 | น้ำเงิน | ดี |
| 2.0 – 2.9 | เหลือง | พอใช้ |
| < 2.0 | แดง | ต้องปรับปรุง |

---

### 13.2 แท็บ: Templates (เกณฑ์ประเมิน)

แสดงรายการ Template การประเมิน

**สร้าง Template ใหม่:**
1. กด **"New Template"**
2. กรอก **Template Name**
3. กด **"+"** เพิ่มเกณฑ์การประเมิน:
   - ชื่อเกณฑ์
   - คำอธิบาย
   - น้ำหนัก (%) — **รวมทุกเกณฑ์ต้องเท่ากับ 100%**
4. กด **"Create Template"**

**เกณฑ์เริ่มต้นในระบบ:**
| เกณฑ์ | น้ำหนัก |
|---|---|
| Price Competitiveness | 30% |
| Quality | 25% |
| Delivery | 20% |
| Service | 15% |
| ESG Compliance | 10% |

---

### 13.3 ประเมิน Supplier

1. กด **"Score Supplier"**
2. เลือก **Supplier** (เฉพาะที่อนุมัติแล้ว)
3. เลือก **Template** (เกณฑ์ที่จะใช้)
4. กรอก **Period** เช่น Q1 2026
5. ให้คะแนนแต่ละเกณฑ์ (1–5) โดยเลื่อน Slider:

| คะแนน | ความหมาย | สี |
|---|---|---|
| 5 | Excellent / ดีเยี่ยม | เขียว |
| 4 | Good / ดี | น้ำเงิน |
| 3 | Average / ปานกลาง | เหลือง |
| 2 | Below Average / ต่ำกว่าเกณฑ์ | ส้ม |
| 1 | Poor / แย่ | แดง |

6. (ไม่บังคับ) กรอก Comment สำหรับแต่ละเกณฑ์
7. กรอก Notes ภาพรวม
8. กด **"Submit Evaluation"**

### 13.4 เปรียบเทียบ Supplier (Comparison)

1. กด **"Compare"**
2. เลือก Template
3. ระบบแสดง:
   - **Auto-Ranking** — จัดอันดับ Supplier จากคะแนนสูงสุด พร้อมเหรียญ 🥇🥈🥉

## 16. การอนุมัติ Supplier

เมนู **Supplier Approvals** — สำหรับ Admin เท่านั้น

### 16.1 หน้าหลัก

**การ์ดสรุป:**
| การ์ด | สี | ความหมาย |
|---|---|---|
| รอตรวจสอบ | เหลือง | Supplier ที่ส่งข้อมูลมาแล้ว รอตรวจสอบ |
| กำลังตรวจสอบ | น้ำเงิน | อยู่ระหว่างพิจารณา |
| ทั้งหมด | เทา | จำนวนรวมทั้งสองสถานะ |

**ตาราง:**
| คอลัมน์ | ความหมาย |
|---|---|
| บริษัท | ชื่อบริษัท Supplier |
| Tax ID | เลขภาษี |
| ผู้ลงทะเบียน | ชื่อหรืออีเมลผู้ที่ลงทะเบียน |
| สถานะ | สถานะปัจจุบัน (ภาษาไทย) |
| วันที่ | วันที่ลงทะเบียน |
| การดำเนินการ | ปุ่มตรวจสอบ / อนุมัติ |

---

### 16.2 ขั้นตอนการอนุมัติ Supplier

1. กดปุ่ม **"ตรวจสอบ"** (ไอคอนตา 👁) เพื่อเปิดรายละเอียด
2. ตรวจสอบข้อมูล 3 ส่วน:
   - **ข้อมูลบริษัท** — ชื่อ, Tax ID, ที่อยู่, เบอร์โทร, อีเมล
   - **ข้อมูลติดต่อ** — รายชื่อผู้ติดต่อ
   - **เอกสาร** — ดาวน์โหลดและตรวจสอบไฟล์แต่ละรายการ
3. เมื่อพิจารณาเสร็จ เลือก:

**กรณีอนุมัติ:**
- กด **"อนุมัติ"** (ปุ่มสีเขียว)
- ระบบจะ:
  - เปลี่ยนสถานะ Supplier เป็น Approved
  - เปิดใช้งานบัญชีผู้ใช้ของ Supplier
  - ส่งการแจ้งเตือนในระบบ
  - ส่งอีเมลแจ้ง Supplier (ถ้าเปิดใช้งาน)

**กรณีปฏิเสธ:**
- กรอก **"เหตุผลกรณีปฏิเสธ"** (บังคับ)
- กด **"ปฏิเสธ"** (ปุ่มสีแดง)
- ระบบจะ:
  - เปลี่ยนสถานะ Supplier เป็น Rejected
  - ส่งการแจ้งเตือนพร้อมเหตุผล
  - ส่งอีเมลแจ้ง Supplier (ถ้าเปิดใช้งาน)

## 17. ระบบแจ้งเตือน

**ไอคอนกระดิ่ง 🔔** ที่มุมบนขวาของหน้า

### 17.1 การใช้งาน

- **ตัวเลขสีแดง** บนไอคอน = จำนวนการแจ้งเตือนที่ยังไม่ได้อ่าน
- คลิกไอคอนเพื่อดูรายการแจ้งเตือนล่าสุด (20 รายการ)
- **คลิกที่แจ้งเตือน** เพื่ออ่านและไปยังหน้าที่เกี่ยวข้อง
- กด **"อ่านทั้งหมด"** เพื่อทำเครื่องหมายว่าอ่านแล้วทั้งหมด

### 17.2 ประเภทการแจ้งเตือน

| เหตุการณ์ | ผู้รับแจ้งเตือน |
|---|---|
| Supplier ลงทะเบียนใหม่ | Admin ทุกคน |
| Supplier ได้รับการอนุมัติ | Supplier นั้น |
| Supplier ถูกปฏิเสธ | Supplier นั้น (พร้อมเหตุผล) |

> การแจ้งเตือนอัปเดตแบบ **Real-time** โดยไม่ต้อง Refresh หน้า

## 18. คำถามที่พบบ่อย

**Q: ลืมรหัสผ่าน ต้องทำอย่างไร?**
A: ติดต่อผู้ดูแลระบบ (Admin) เพื่อรีเซ็ตรหัสผ่านให้

---

**Q: ทำไม Supplier ถึงเข้าสู่ระบบไม่ได้หลังลงทะเบียน?**
A: ต้องรอผู้ดูแลระบบอนุมัติก่อน เมื่ออนุมัติแล้วจึงจะเข้าสู่ระบบได้

---

**Q: สร้าง RFQ แล้ว Publish ไม่ได้?**
A: ตรวจสอบว่าได้เชิญ Supplier อย่างน้อย 1 รายแล้วในแท็บ "Invited Suppliers"

---

**Q: ทำไม Supplier บางรายไม่ปรากฏในรายการเชิญ RFQ?**
A: เฉพาะ Supplier ที่มีสถานะ **Approved** เท่านั้นที่จะปรากฏในรายการ

---

**Q: ไฟล์เอกสารที่อัปโหลดได้สูงสุดเท่าไหร่?**
A: สูงสุด **10 MB** ต่อไฟล์ รองรับ PDF, JPG, PNG, DOC, DOCX

---

**Q: ใครสามารถอนุมัติ Award ได้?**
A: เฉพาะ **Admin** และ **Approver** เท่านั้น

---

**Q: จะเปลี่ยนบทบาทผู้ใช้ได้อย่างไร?**
A: Admin สามารถจัดการบทบาทผู้ใช้ได้ที่เมนู **Admin Settings → Users**

---

**Q: ประมูล (Bidding) หยุดเองได้ไหม?**
A: ใช่ ระบบจะปิดการประมูลอัตโนมัติเมื่อถึงเวลา End Time ที่กำหนด หรือผู้ดูแลสามารถกด Close Auction ได้เลย

---

**Q: คะแนน ESG คำนวณจากอะไร?**
A: คะแนน ESG รวม = ค่าเฉลี่ยของ Environmental + Social + Governance (แต่ละด้าน 0–100)

```
draft → submitted → review → approved
                          ↘ rejected → draft

scheduled → active → closed

```
pending → approved
        ↘ rejected
        ↘ revise → pending

```
pending → selected → ready_for_po → (create award)
        ↘ rejected
```
